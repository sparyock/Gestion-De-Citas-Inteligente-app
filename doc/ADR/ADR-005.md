# 5. Diseño e Implementación del ADR

## 5.1 Contexto

El sistema requiere validar:

- fecha obligatoria  
- hora válida  
- disponibilidad del doctor  
- duplicidad de turnos  

---

## 5.2 Decisión

Implementar validación en:

Service Layer

Usando:

- @Valid  
- reglas de negocio  
- validaciones personalizadas  

---

## 5.3 Diseño de la Solución

```
Controller
   │
   ▼

Service
   │
   ├── Validación de reglas
   │
   ▼

Repository
```

---

## 5.4 Implementación

```java
public Turno save(Turno turno) {

    if(turno.getFecha() == null){
        throw new ValidationException(
            "La fecha es obligatoria"
        );
    }

    if(turno.getHora() == null){
        throw new ValidationException(
            "La hora es obligatoria"
        );
    }

    return repository.save(turno);

}

```

---

## 6.3 Consecuencias

Positivas:

- Mayor seguridad  
- Menos errores

Negativas:

# Decisión Final

Se implementará validación de reglas de negocio a nivel de servicio para garantizar la integridad de los datos en el sistema de Gestión de Citas Inteligente.
