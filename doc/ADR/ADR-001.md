## 1. Identificación de Patrones

### 1.1 Patrones Actualmente Implementados

| Patrón               | Ubicación                            | Descripción                                        |
| -------------------- | ------------------------------------ | -------------------------------------------------- |
| Repository Pattern   | TurnoRepository, UserRepository      | Abstracción de acceso a datos usando JpaRepository |
| DTO Pattern          | TurnoDTO, UserDTO                    | Transferencia de datos entre capas                 |
| Service Layer        | TurnoService, UserService            | Capa de lógica de negocio separada                 |
| Dependency Injection | @Autowired, @RequiredArgsConstructor | Inyección de dependencias mediante Spring          |

### 1.2 Patrones Ausentes Críticos

| Patrón Faltante      | Justificación                                      |
| -------------------- | -------------------------------------------------- |
| Generic Base Service | Existe duplicación de lógica CRUD en los servicios |
| Mapper Pattern       | Conversión manual entre Entity y DTO               |
| Custom Exceptions    | Manejo genérico de errores con RuntimeException    |
| Validation Pattern   | Falta validación en los endpoints                  |

## 3. Identificación de Antipatrones

### 3.1 Antipatrón: Copy-Paste Programming

**Ubicación:** `TurnoServiceImpl.java`

```java
 @Override
public List<TurnoDTO> findAll(){
    return turnoRepository.findAll()
        .stream()
        .map(this::mapToDTO)
        .toList();
}
```

**Problema:** 

- Código repetido en diferentes servicios
- Dificulta el mantenimiento
- Incrementa riesgo de errores

--- 

### 3.2 Antipatrón: Magic Strings

**Ubicación:**  `Servicios`

```java

throw new RuntimeException("Turno no encontrado");

```

**Problemas:**

- Falta de manejo adecuado de errores
- No se pueden diferenciar tipos de excepciones
- Complica el debugging

--- 
### 3.3 Antipatrón: Lógica repetida en servicios

Los métodos CRUD se implementan manualmente en cada servicio:

```java 
findAll()
findById()
save()
delete()
```

**Esto genera:**

- duplicación de código
- baja mantenibilidad
- dificultad para escalar el sistema

----

### 4. Segmentos de Código con Malas Prácticas

**Mapeo manual repetitivo**

Ejemplo:

```java
private TurnoDTO mapToDTO(Turno turno){

    TurnoDTO dto = new TurnoDTO();

    dto.setId(turno.getId());
    dto.setFecha(turno.getFecha());
    dto.setDoctor(turno.getDoctor());

    return dto;
}
```

**Problemas:**

- código repetitivo
- difícil de mantener
- riesgo de olvidar campos


## 5. Diseño e Implementación del ADR
### 5.1 Contexto


El sistema maneja múltiples entidades:

- Turnos
- Usuarios
- Doctores
- Especialidades

Todas requieren operaciones CRUD.
Actualmente estas operaciones se repiten en cada servicio.


---
### 5.2 Decisión

Se decide implementar un Generic Base Service utilizando genéricos.

**Convención de genéricos:**

| Genérico | Significado     | Ejemplo  |
| -------- | --------------- | -------- |
| T        | Tipo de entidad | Turno    |
| D        | Tipo de DTO     | TurnoDTO |

Esto permitirá reutilizar la lógica CRUD en múltiples servicios.

### 5.4 Implementación Propuesta

**Interfaz Base Genérica**

``` Java
public interface IBaseService<T,D>{

    List<D> findAll();

    D findById(Long id);

    D save(D dto);

    void delete(Long id);
}

```

**Implementación Base**

```Java
@RequiredArgsConstructor
public abstract class BaseServiceImpl<T,D> implements IBaseService<T,D>{

    protected final JpaRepository<T,Long> repository;

    @Override
    public List<T> findAll(){
        return repository.findAll();
    }

    @Override
    public void delete(Long id){
        repository.deleteById(id);
    }
}
```

## 6. Impacto sobre el Sistema

### 6.1 Beneficios

| Beneficio                     | Descripción                                    | Impacto |
| ----------------------------- | ---------------------------------------------- | ------- |
| Reducción de código duplicado | Se centraliza la lógica CRUD                   | Alto    |
| Mayor mantenibilidad          | Cambios en un solo lugar                       | Alto    |
