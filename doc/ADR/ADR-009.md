## 2. Análisis del Proyecto y ADRs Potenciales

### 2.1 Estado Actual de la Arquitectura

El sistema tiene una arquitectura en capas compuesta por:

- Controllers  
- Services  
- Repositories  
- Entities  
- DTOs

## 3. Identificación de Antipatrones

### 3.1 Antipatrón: Improper Resource Identification

**Ubicación:** `UserController.java`

Actualmente el método recibe el parámetro directamente.

Ejemplo:

```java
public UserDTO getUser(Long id){
    return userService.findById(id);
}
```
**Problema:**

- No se utiliza la URL para identificar recursos
- Difícil comprensión de los endpoints
- No se sigue el estándar REST

---

### 3.2 Antipatrón: Non-RESTful Endpoint Design

Los endpoints no siguen la estructura estándar REST.

Ejemplo:

GET /getUser?id=1

**Problemas:**

- Mala organización de endpoints
- Falta de claridad en las rutas
- Dificultad para mantenimiento

---
### 3.3 Antipatrón: Inconsistent URL Structure

El sistema utiliza diferentes formas para identificar recursos.

**Esto genera:**

- Inconsistencia en la API
- Confusión para desarrolladores
- Dificultad en integración con frontend

---
## 4. Segmentos de Código con Malas Prácticas

Ejemplo actual:

```java
@GetMapping("/getUser")
public UserDTO getUser(Long id){

    return userService.findById(id);

}

```

**Problemas:**

- No utiliza parámetros en la ruta
- No sigue estándar REST
- Dificulta mantenimiento del sistema

---
## 5. Diseño e Implementación del ADR
### 5.1 Contexto

El sistema maneja múltiples entidades:

- Usuarios
- Turnos
- Doctores
- Especialidades

Estas entidades deben ser accesibles mediante endpoints claros y organizados que sigan estándares REST.

---
### 5.2 Decisión

Se decide utilizar la anotación @PathVariable para identificar recursos mediante la URL en los endpoints del sistema.

**Esto permitirá:**

- Cumplir con estándares REST
- Mejorar organización de rutas
- Facilitar mantenimiento del sistema

---

### 5.3 Diseño de la Solución

Arquitectura propuesta:
```

└── UserController

   │

└── UserService

Base de Datos
```

---

### 5.4 Implementación Propuesta

``` Java
@GetMapping("/usuarios/{id}")
public UserDTO getUser(
        @PathVariable Long id){

    return userService.findById(id);

}
```
