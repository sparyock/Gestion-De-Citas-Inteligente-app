## 3. Identificación de Antipatrones

### 3.1 Antipatrón: Works on My Machine

**Ubicación:** `docker/.gitkeep`

```
docker/
└── .gitkeep  ← archivo vacío, sin docker-compose.yml
```

**Problema:**

- Sin orquestación, cada desarrollador configura el entorno a su manera
- Los puertos, usuarios y contraseñas varían por máquina
- La demo ante el profesor depende de que "todo esté bien configurado"

---

### 3.2 Antipatrón: Manual Environment Setup

**Ubicación:** README.md (proceso manual documentado)

```bash
# Proceso actual sin Docker Compose
# Paso 1: instalar PostgreSQL manualmente
# Paso 2: crear base de datos manualmente
# Paso 3: configurar application.properties manualmente
# Paso 4: levantar cada servicio por separado
```

**Problemas:**

- Proceso repetitivo y propenso a errores
- No reproducible entre diferentes máquinas
- Tiempo de setup elevado para nuevos integrantes

## 5. Diseño e Implementación del ADR

### 5.1 Contexto

El proyecto tiene una carpeta `docker/` vacía y el README documenta el uso de `docker-compose up`, pero ese archivo no existe. Cada integrante levanta los servicios de forma manual e inconsistente.

---

### 5.2 Decisión

Se decide crear el archivo `docker-compose.yml` en la carpeta `docker/` que levante PostgreSQL y pgAdmin con un solo comando, garantizando que el entorno sea idéntico para todos los integrantes del equipo.

| Servicio  | Imagen          | Puerto | Responsabilidad                  |
| --------- | --------------- | ------ | -------------------------------- |
| postgres  | postgres:16     | 5432   | Base de datos relacional         |
| pgadmin   | dpage/pgadmin4  | 5050   | Interfaz visual para la BD       |

---

### 5.3 Diseño de la Solución

```
┌────────────────────────────────────────────┐
│            ARQUITECTURA PROPUESTA          │
├────────────────────────────────────────────┤

Un solo comando:
$ docker-compose up -d

              │
              ▼

┌─────────────────────────────┐
│       citas-network         │
│                             │
│  ┌──────────┐  ┌─────────┐  │
│  │ postgres │  │pgadmin  │  │
│  │  :5432   │  │  :5050  │  │
│  └──────────┘  └─────────┘  │
│                             │
└─────────────────────────────┘

              │
              ▼

users-service   → conecta a postgres:5432/usersdb
turnos-service  → conecta a postgres:5432/turnosdb

└────────────────────────────────────────────┘
```

---

### 5.4 Implementación Propuesta

**docker/docker-compose.yml**

```yaml
version: '3.8'

services:

  postgres:
    image: postgres:16
    container_name: citas-postgres
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: admin1234
      POSTGRES_DB: usersdb
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    networks:
      - citas-network

  pgadmin:
    image: dpage/pgadmin4
    container_name: citas-pgadmin
    environment:
      PGADMIN_DEFAULT_EMAIL: admin@citas.com
      PGADMIN_DEFAULT_PASSWORD: admin1234
    ports:
      - "5050:80"
    depends_on:
      - postgres
    networks:
      - citas-network

volumes:
  postgres_data:

networks:
  citas-network:
    driver: bridge
```

**Comando para levantar el sistema**

```bash
cd docker
docker-compose up -d
```

**Accesos disponibles**

```
PostgreSQL → localhost:5432
pgAdmin   → localhost:5050
```

## 6. Impacto sobre el Sistema

### 6.1 Beneficios

| Beneficio                    | Descripción                                               | Impacto |
| ---------------------------- | --------------------------------------------------------- | ------- |
| Entorno reproducible         | Todos los integrantes tienen el mismo entorno             | Alto    |
| Setup en un comando          | docker-compose up -d levanta todo el sistema              | Alto    |
| Persistencia de datos        | El volumen Docker conserva los datos entre reinicios      | Alto    |
| Interfaz visual para la BD   | pgAdmin disponible en localhost:5050                      | Medio   |

---

### 6.2 Métricas Esperadas

```
ANTES

Setup por integrante → 20-30 minutos de configuración manual
Errores de entorno   → frecuentes por diferencias de configuración
Demo del proyecto    → riesgo alto de fallar por configuración


DESPUÉS

Setup por integrante → 2 minutos (docker-compose up -d)
Errores de entorno   → eliminados (mismo entorno para todos)
Demo del proyecto    → entorno estable y reproducible
```

- Tiempo de setup: 30 min → 2 min
- Riesgo de fallos por entorno: Alto → Bajo

---

### 6.3 Riesgos

| Riesgo                        | Probabilidad | Impacto | Mitigación                              |
| ----------------------------- | ------------ | ------- | --------------------------------------- |
| Docker Desktop no instalado   | Media        | Medio   | Documentar prerequisito en README       |
| Puerto 5432 ocupado           | Baja         | Bajo    | Cambiar el puerto externo en el compose |
| Conflicto de volúmenes        | Baja         | Bajo    | docker-compose down -v para limpiar     |
