## 2. Análisis del Proyecto y ADRs Potenciales

### 2.1 Estado Actual de la Arquitectura

El sistema de Gestión de Citas Inteligentes cuenta con un frontend y un backend que se ejecutan en diferentes puertos.

Por ejemplo:

- Frontend: localhost:3000  
- Backend: localhost:8080  

# Diagrama de arquitectura actual:

```
┌─────────────────────────────────────────────┐
│ ARQUITECTURA ACTUAL │
├─────────────────────────────────────────────┤

Frontend Layer
└── React App (localhost:3000)

Backend Layer
└── Spring Boot API (localhost:8080)

Comunicación
└── Bloqueada por política de seguridad (CORS)

└─────────────────────────────────────────────┘
```

**Problema detectado:**

El navegador bloquea las solicitudes del frontend al backend debido a restricciones de seguridad conocidas como CORS.

## 3. Identificación de Antipatrones

### 3.1 Antipatrón: Missing CORS Configuration

**Ubicación:** `TurnoController.java`

Actualmente el backend no permite solicitudes desde otros orígenes.

Ejemplo:

```java
@RestController
public class TurnoController {
}

```

**Problema:**

El backend no permite solicitudes desde otros orígenes.

**Esto genera:**

- Bloqueo de solicitudes HTTP  
- Fallo en integración frontend-backend  
- Imposibilidad de consumir la API  

---
### 3.2 Antipatrón: Cross-Origin Request Blocked

Cuando el frontend intenta consumir la API, el navegador genera el siguiente error:

```java
Access to fetch at 'http://localhost:8080'
from origin 'http://localhost:3000'
has been blocked by CORS policy
```

**Problemas:**

- El sistema no funciona correctamente
- El usuario no puede interactuar con el sistema
- Se dificulta el desarrollo y pruebas

---

### 3.3 Antipatrón: Falta de Configuración de Seguridad en Comunicación

El sistema no define reglas de seguridad para permitir o restringir accesos desde diferentes orígenes.

**Esto genera:**

- Riesgo de acceso no controlado
- Problemas de integración
- Falta de control sobre solicitudes externas

## 5. Diseño e Implementación del ADR
### 5.1 Contexto

El sistema utiliza un frontend que consume los servicios del backend mediante peticiones HTTP.

Para permitir esta comunicación es necesario configurar CORS.

---
### 5.2 Decisión

Se decide habilitar CORS en los controladores del sistema utilizando la anotación:
```java
@CrossOrigin
```
---
### 5.3 Diseño de la Solución

Arquitectura propuesta:

```
┌────────────────────────────────────────────┐
│            ARQUITECTURA PROPUESTA          │
├────────────────────────────────────────────┤

Frontend Layer
└── React App

Backend Layer
└── Spring Boot API

Comunicación
└── Permitida mediante @CrossOrigin

└────────────────────────────────────────────┘
```
---
### 5.4 Implementación Propuesta

``` Java
@CrossOrigin(origins = "http://localhost:3000")
@RestController
public class TurnoController {
}
```
